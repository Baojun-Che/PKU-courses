# 数据清洗报告（方法说明版）

本报告说明 `download_data.py` 与 `clean_data.py` 的数据下载和清洗逻辑。运行 `clean_data.py` 后，程序还会在 `data/cleaned/data_cleaning_report.md` 中自动生成一份带有实际资产数量、删除原因和异常值数量的报告。

## 1. 数据下载

下载脚本为 `download_data.py`。它读取 `data/sp500_top80_universe.csv` 中的股票池，默认优先使用 `YahooTicker` 列；如果没有该列，则使用 `Ticker` 列，并自动把 `BRK.B`、`BF.B` 这类代码转换成 Yahoo 风格的 `BRK-B`、`BF-B`。

下载区间默认为：

```text
2015-01-01 至 2024-12-31
```

脚本不使用 `yfinance`，而是通过 `requests` 直接下载日频 OHLCV 数据。下载结果保存为长表：

```text
data/raw_prices/all_daily_prices.csv
```

其中最重要的价格列是：

```text
AdjClose
```

后续清洗和收益率构造只使用 adjusted close。

## 2. 统一交易日历

清洗脚本为 `clean_data.py`。它先将长表价格数据转换为宽表：

```text
行：Date
列：Ticker
值：AdjClose
```

为了让所有资产对齐到同一个交易日历，程序计算每个日期上有有效价格的资产比例。默认情况下，如果某个日期至少有 80% 的下载资产存在非缺失 adjusted close，则该日期被纳入共同交易日历。

这个规则可以过滤掉明显异常的日期，例如只有极少数资产有价格记录的日期。

## 3. 删除缺失值太多的资产

在共同交易日历上，程序会为每只资产计算：

- 有效价格数量；
- 缺失价格数量；
- 缺失价格比例；
- 第一个有效价格日期；
- 最后一个有效价格日期；
- 最大连续缺失交易日数量。

默认删除规则为：

```text
价格缺失比例 > 5%                      删除
第一个有效价格晚于起始日 31 天以上       删除
最后一个有效价格早于结束日 10 天以上      删除
最大连续缺失交易日数量 > 5               删除
```

删除明细会保存到：

```text
data/cleaned/asset_cleaning_diagnostics.csv
```

## 4. 退市资产、停牌资产和历史太短资产的处理

### 退市资产

如果某资产无法下载，或者在样本结束日前很早就没有有效价格，脚本会将其排除。这样可以避免把明显不完整的价格序列放进最终收益率矩阵。

不过需要注意：如果股票池来自“当前 S&P 500 市值前 80”，那么样本存在 survivorship bias。也就是说，它没有包含 2015 到 2024 年期间曾经是大市值公司、但后来被并购、退市或跌出指数的公司。如果研究目标需要严格控制退市偏差，应使用 CRSP、Compustat、Bloomberg、Refinitiv 等包含历史成分股和退市收益的数据源。

### 停牌资产

对于短暂停牌或短数据缺口，脚本默认最多向前填充 2 个交易日。超过阈值的长缺口会导致资产被删除。

这样做的目的，是避免长时间停牌资产在样本中被人为处理成长期 0 收益。

### 历史太短资产

如果某资产的第一个有效价格距离起始日太远，说明它可能 IPO 较晚、上市历史不足或数据覆盖不足。默认情况下，这类资产会被删除，以保证最终面板中的资产拥有尽量一致的历史长度。

## 5. 缺失值处理

缺失值处理原则如下：

1. 价格小于等于 0 的记录视为无效价格，设为缺失值；
2. 先删除缺失比例过高、历史太短、结束太早或有长缺口的资产；
3. 对剩余资产的短价格缺口做有限向前填充，默认最多 2 个交易日；
4. 不使用向后填充，因为向后填充会使用未来信息；
5. 构造收益率后，如果某个日期仍存在任一资产的缺失收益率，默认删除该日期；
6. 最终 `cleaned_returns.csv` 默认是无缺失值的平衡收益率面板。

## 6. 收益率构造方法

收益率使用 adjusted close 构造，公式为：

```text
r[t, i] = (p[t+1, i] - p[t, i]) / p[t, i]
        = p[t+1, i] / p[t, i] - 1
```

其中：

```text
p[t, i] = 股票 i 在日期 t 的 adjusted close
```

`cleaned_returns.csv` 中的 `Date` 表示公式中的 `t`，也就是持有期开始日。因此，每一行收益率表示从当天 adjusted close 到下一交易日 adjusted close 的简单收益率。

最后一个价格日期没有 `p[t+1]`，因此会被删除。

## 7. 异常值处理

异常值处理分两步：

### 7.1 硬阈值过滤

如果某个单日向前收益率满足：

```text
|r[t, i]| > 50%
```

则默认认为它是高度可疑值，可能来自数据错误、未被正确调整的公司行为或其他异常记录。脚本会将其设为缺失值，并在最终日期过滤时处理。

### 7.2 分资产缩尾处理

对每只资产分别做 winsorization。默认分位点为：

```text
下分位点：0.1%
上分位点：99.9%
```

也就是说，每只股票自己的极端收益率会被压缩到该股票自身的 0.1% 和 99.9% 分位点以内。

异常值处理统计会保存到：

```text
data/cleaned/outlier_report.csv
```

## 8. 清洗前后资产数量报告

运行 `clean_data.py` 后，程序会自动报告以下数量：

- 初始股票池资产数量；
- 下载成功资产数量；
- 进入价格矩阵的资产数量；
- 清洗后保留资产数量；
- 被删除资产数量；
- 删除原因汇总；
- 最终收益率矩阵的日期数量；
- 最终收益率是否仍有缺失值。

这些信息会保存到：

```text
data/cleaned/cleaning_summary.json
data/cleaned/data_cleaning_report.md
```

## 9. 最终输出

最终核心文件为：

```text
data/cleaned/cleaned_returns.csv
```

它是一个宽表收益率矩阵：

```text
行：Date
列：Ticker
值：forward simple return
```

该文件可以直接用于后续组合优化、协方差估计、因子回归、机器学习预测或风险模型构建。
